# GoldSapa ML — Предиктивное планирование производства

## О проекте

Система прогнозирования ежедневных продаж для хлебозавода **Gold Sapa** (Алматы, Казахстан). Цель — **автоматически формировать утреннюю загрузку мощностей (печей)** до прихода основного заказа от торговых представителей.

### Бизнес-контекст

Каждое утро технолог вручную решает, сколько какого продукта загрузить в печи. Заказы от клиентов приходят только к 16:00–18:00, но печи нужно запускать с утра — иначе простой мощностей.

**Модель предсказывает**: сколько штук каждого товара будет продано завтра → технолог знает, чем загрузить печи утром.

| Время | Что происходит |
|---|---|
| **Утро** | Технолог смотрит прогноз от ML → загружает печи |
| День | Торговые представители собирают заказы |
| 16:00 | Предварительный заказ на производство |
| 18:00 | Основной заказ на производство |
| Ночь | Производство выполняет оставшийся заказ |

## Данные

### Источник
1С через OData API: `https://goldsapa.24c.kz/goldsapa/odata/standard.odata`

| Данные | OData endpoint | Описание |
|--------|---------------|----------|
| Продажи | `AccumulationRegister_РеализацияТМЗ_RecordType` | Фактические продажи (кол-во, сумма, дата, товар, склад) |
| Справочник | `Catalog_Номенклатура` | Наименования товаров (Ref_Key, Description, Parent_Key) |

### Загрузка
- Продажи грузятся **по неделям** через `$filter` по `Period` (не `$top/$skip`)
- Фильтр: только `Recorder_Type = РеализацияТоваровУслуг`
- Resume: если недельный чанк уже скачан — пропускается

### Объём данных (на 26.03.2026)

| Метрика | Значение |
|---------|----------|
| Период | 2024-01-03 — 2026-03-26 |
| Сырых записей | 331,781 |
| Дневных агрегатов | 79,514 |
| Товаров | 166 |
| Складов | 4 |
| Общая выручка | 1.3 млрд тг |

## Модели

Все модели: **CatBoostRegressor**. Train: 2024–2025, Val: 2026.

| Модель | MAE (шт) | MAPE | R² | Ключевые признаки |
|--------|----------|------|-----|-------------------|
| Baseline | 82.77 | 165.3% | 0.554 | Календарь + товар/склад |
| **V2** ✅ | **55.90** | 109.1% | **0.754** | + лаги, MA, цена, погода |
| V3 | 60.54 | 54.1% | 0.632 | + log-transform, EWMA, категории |

**Лучшая модель: V2** (минимальный MAE = 55.90 шт).

V3 с лог-трансформацией улучила MAPE (54% vs 109%), но ухудшила MAE — для задачи загрузки печей важнее абсолютная точность в штуках.

### Top Feature Importance (V2)
1. `avg_price` — 30% (цена за единицу)
2. `ma_7` — 22% (скользящее среднее 7 дней)
3. `lag_7` — 14.4% (продажи неделю назад)
4. `ma_14` — 13.1%

**Вывод**: модель больше всего опирается на недавнюю историю продаж конкретного товара. Погода и праздники почти не влияют (хлеб покупают всегда).

## Быстрый старт

### 1. Клонирование и установка

```bash
git clone https://github.com/aleka07/GoldSapa_ML.git
cd GoldSapa_ML
uv sync          # создаёт .venv и ставит все зависимости из uv.lock
```

> **Примечание:** данные (`data/`) не хранятся в репозитории — попросите их у автора и положите в корень проекта.

### 2. Запуск пайплайна

```bash
# Активировать окружение
source .venv/bin/activate

# Шаг 1: Выгрузка продаж
python -m src.extract --date-from 2024-01-01 --date-to 2026-03-26

# Справочник номенклатуры
python -m src.extract --nomenclature

# Шаг 2: Агрегация
python -m src.aggregate

# Аналитика
python -m src.analytics

# Обучение
python -m src.model       # baseline
python -m src.model_v2    # лучшая модель
python -m src.model_v3    # экспериментальная
```

### 3. Добавление зависимостей

```bash
uv add <package>     # добавляет в pyproject.toml + обновляет uv.lock
```
