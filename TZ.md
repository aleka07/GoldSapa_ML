# Техническое задание
## Система предиктивного планирования производства — Gold Sapa

### 1. Контекст и проблема

Компания производит и реализует пищевую продукцию. Ежедневно технолог вручную определяет объём производства на следующий день, опираясь исключительно на личный опыт. Этот процесс не формализован, зависит от одного человека и не учитывает накопленную статистику продаж системно.

**Цель:** автоматизировать формирование предварительного заказа на производство на основе предиктивной аналитики.

---

### 2. Текущий бизнес-процесс

| Время | Событие |
|---|---|
| Утро | Технолог формирует заказ на производство вручную |
| День | Торговые представители собирают заказы от клиентов |
| 16:00 | Предварительный заказ на производство (на основе накопленных заказов) |
| 18:00 | Основной заказ на производство |
| до 21:00 | Операторы принимают заказы по телефону |
| Ночь | Производство выполняет заказ |
| Утро следующего дня | Готовая продукция |

**Факторы которые технолог учитывает интуитивно:**
- День недели
- Температура воздуха в городе
- Праздники и выходные дни
- Прочие неформализованные факторы

---

### 3. Источники данных

**Используем:** Регистр накопления `РеализацияТМЗ` — фактические продажи по номенклатуре, складу, дате.

**Не используем:** Документ производства — данные вторичны, вносятся постфактум и не отражают спрос.

**Внешние данные:**
- Погода: исторический архив + прогноз на завтра (Open-Meteo API, бесплатно)
- Праздники: производственный календарь Республики Казахстан

---

### 4. Задача системы

На основе истории продаж и внешних факторов система должна к **началу рабочего дня** автоматически формировать рекомендованный объём производства по каждой позиции номенклатуры на следующий день — до того как технолог приступает к ручному планированию.

### 6. Стек реализации

- **Источник данных:** 1С через OData (`AccumulationRegister_РеализацияТМЗ/RecordType`)
- **Обработка:** Python, pandas
- **Модель:** CatBoostRegressor
- **Погода:** Open-Meteo API
- **Праздники:** библиотека `holidays` (Kazakhstan)
- **Интерфейс:** веб-дашборд или Excel-выгрузка для технолога
